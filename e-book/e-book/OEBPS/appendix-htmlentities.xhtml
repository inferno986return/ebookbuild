Appendix

HTML entities cheatsheet

As mentioned in the Introduction to XHTML chapter, both HTML and XHTML reserve metacharacters so to use a metacharacter literally its entity must be used. Almost all common characters in ASCII and Unicode have entities, but only a comparitive few characters cannot be used literally. I have made my own cheatsheat in the table below:

HTML entities cheatsheet
Mandatory entities (these characters cannot be used literally)
Name entity Number entity Character Name
&amp; &#38; & Ampersand
&lt; &#60; < Less than
&gt; &#61; > Greater than
Punctuation
Name entity Number entity Character Name
&apos; &#39; ' Single straight quote/apostrophe
&quot; &#34; " Double straight quote
&lsquo; &#38; ‘ Single curly left quote
&rsquo; &#38; ’ Single curly right quote/apostrophe
&ldquo; &#38; " Double curly left quote
&rdquo; &#38; " Double curly right quote

Simplified Regular Expressions cheatsheet
Anchors
^ Start of line +
$ End of line +
Character Classes

This cheatsheet is derived from the regular expressions cheatsheet by David Child of AddedBytes.com.2

1. As mentioned previously in the chapter on regular expressions, the metacharacter for returning captures varies. It can be a dollar ($) such as in Atom or a backslash (\) such as in Notepad++.

2. The PDF of the cheatsheet can be downloaded from the old website of the Technical University of Denmark: http://www.cbs.dtu.dk/courses/27610/regular-expressions-cheat-sheet-v2.pdf
